Vue3基本语法

发表于2023-08-09|更新于2023-10-25|前端Vue
|阅读量:

Vue3源码位置：https://unpkg.com/vue@next

Vue3基础语法

动态展示数据，列表数据也能表示

Vue的核心开发模式（计数器案例）

真正开发不把渲染模板放在template这样写，Vue内如果不写template会自动把对应元素（div.#add）写在html内的部分当成渲染模板

Options API

data

data必须是一个函数，必须返回一个对象，data对对象里的变量是实时监听劫持的，Vue2的原理是Object.defineProperty(对象，变量，{})，Vue3的原理是 Proxy代理

methods

methods的方法可以绑定到模板中，可以用this直接访问修改data中返回的对象的属性

所以methods的函数不能写成箭头函数

源码中是遍历了所有函数，并bind绑定的一个Proxy代理对象以后进阶再说，现在先会用

模板语法

mustache双大括号，插值语法

双大括号中不仅仅是data的属性，还可以是JS的一个表达式就是类似

还可以调用methods中的函数，得到的就是返回值

指令

  • v-once ：只渲染一次，改变了就不更新了
  • v-bind：放在属性前可以渲染属性值
  • v-on：绑定事件，前面写的@click是语法糖

事件传递若是没有参数就默认传递对应的event对象，如果有参数还要传event就加$event

修饰符，对时间进行特殊处理

  • stop，阻止事件冒泡，相当于调用event.stopPropagation()
  • prevent，组织元素默认行为

等等……

条件渲染

v-if，v-else，v-else-if满足条件渲染，不满足不渲染，v-show

v-show和v-if区别：

  1. v-show不支持template，没有else分支

  2. v-show条件为false时，元素不会被销毁，本质是修改display属性为none，所以频繁切换用v-show防止过多的回流和重绘

template元素

template元素存在的意义就是包裹一段不知道要不要渲染的html元素，在template元素中进行判断而template本身是不会进入DOM树进行渲染的，就不用用div包裹了，相当于是一个提高性能小优化

v-for

v-for可以遍历可迭代对象，基本语法格式：”item in 数组”，要是索引值也要拿就是：”(item, index) in 数组”

开发时只要写for循环就给每一个元素加一个key属性，具有唯一的属性值，原因设计到vue算法原理，先理解下面的概念

Vnode：虚拟节点，html元素创建出来的，无论是组件还是元素，在Vue中都是Vnode，Vnode本质是一个JS的对象，所以是代码->Vnode Tree -> 真实DOM Tree 这样一个过程

为什么要进行这样看似多此一举的过程？

  1. 跨平台，我有一个虚拟Tree，就不仅仅能渲染到浏览器上，还可以移动端，桌面端，VR等等
  2. 方便Vue进行diff算法，也是为什么v-for要加key的原因

有key的diff算法：在数组中间加减元素时，diff算法会根据key属性尽量的复用修改之前的结点，没有key值就会依次修改后面的元素，两种算法暂时理解成一个接近O（1），一个O（n）但是目前还是不理解怎么能O（1）实现增删改查，学习源码时再说，所以要加key

v-model

用于和表单双向绑定，和用户交互（方便开发，当然也可以用event手动实现双向绑定）

v-model还能绑定textarea， checkbox（单选框绑定bool类型），checkbox（复选框加上value值代表选中绑定什么），radio（单选互斥也要加value），select同理

Options API again

计算属性computed

当模板要用到data里面的数据的复杂逻辑就要用computed

computed是一个对象，里面都是函数，每个函数都是定义的一个data的复杂逻辑，在模板中直接用函数名就能调用

computed和methods区别：

  1. 用的时候加不加括号
  2. computed是有缓存的，一个函数如果内部的变量没有发生改变那么调用就直接返回上次的返回值，提高了性能

watch

监听data的数据变化：也是一个对象，内部是必须以变量名字命名的函数，监听这个变量，每次变化会传入两个参数，一个新值一个旧值（对象是Proxy）

Composition API

相比于vue2的optionsAPI的优点：同一个变量的操作会放到一起

setup函数

在开发中要符合单向数据流的规则，就是父组件给子组件传递的信息只能用不能修改，要修改也得用子组件相父亲传递事件的方式，用readonly函数修饰可以强制这一点

setup的语法糖，能直接把主要逻辑之外的代码都删掉（不好描述看代码）
