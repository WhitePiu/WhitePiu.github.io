组件化开发

发表于2023-08-13|更新于2023-10-25|前端Vue
|阅读量:

注册全局组件

每个组件都有自己的逻辑，每个组件创建也可以定义各种各样的Option API

全局组件特点：创建好也可以用在其他组件的template中

一般开发为了减少打包的大小，只用局部组件，很少全局

局部组件的注册：

组件通信

父组件 -> 子组件：父组件通过属性传递，子组件通过props对象成员接收

props数组语法：不能进行数据类型的验证，没有默认值

props对象语法：解决数组对象的弊端，写起来繁琐一点（常用）

如果默认值是一个函数，那么default必须是一个函数，返回值为一个对象

子组件 -> 父组件：一般发生事件会传递，我们需要让子组件发出一个自定义事件好让父组件监听到完成修改父组件的内容，达到子组件向父组件传递信息的效果

如果父组件给子组件添加了属性但是子组件没有接收，那么会自动传到子组件根元素的attribute上（多个根就显式绑定v-bind=”￥attrs“）

组件插槽slot

因为即使是template，也只是让内容更灵活，里面的元素种类都是固定的，组件插槽就是解决这个问题，让元素也由用户决定，变得更灵活

插槽名用name的属性值表示，父组件用v-slot匹配对应的插槽名：

作用域插槽

slot自定义的属性，在父组件可以用v-slot:default=”props”这样的方式来接收，props是一个对象，里面存放的就是自定义的各种属性，这样父组件就能用子组件的data（）中的东西了

非父子组件通信

Provide

祖先组件用于放数据的Options API

Inject

子组件用于接收数据的Options API

使用第三方库：事件总线event-bus

可以全局创建事件，全局监听

生命周期

beforeCreate -> created -> beforeMount -> mounted -> beforeUpdate -> updated -> beforeUnmount -> unmounted

生命周期函数

ref

用于获取元素获取组件实例，既然用vue声明式开发就不能用原生JS的DOM操作

动态组件

用于显示不同的组件，可以用v-if来进行逻辑判断切换，但是如果要来回切换的组件太多，就会导致写一堆分支，动态组件因此而生

keep-alive

在component外包裹一层keep-alive让这个组件不被销毁
