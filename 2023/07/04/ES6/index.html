箭头函数

箭头函数没有原型不绑定this，arguments属性，不能作为构造函数使用（new）

箭头函数是函数表达式写法的特殊写法

箭头函数的简写

  • 如果箭头函数只有一个参数，那么小括号可以省略
  • 如果函数只有一行执行代码（不能带return），那么可以省略大括号
  • 如果执行了第二步的省略，那么函数自动返回这一行代码的执行结果
  • 默认返回值是一个对象的话，这个对象必须加（）

函数增强

函数对象的属性：

  • name：函数名称
  • length：函数参数个数（定义的，且不算剩余参数，有默认值的参数也不算）

函数的arguments：所有的传进去的参数都会传进arguments类数组对象中

类数组对象转换为数组：

箭头函数不绑定arguments ，而且没有显式原型不绑定super，如果有同this一样从上层作用域找

函数剩余参数

语法：该参数前加三个点，必须写到最后

ES6用来代替arguments的东西，相当于把前面的参数删掉，并转换为数组的arguments

纯函数

定义：

  1. 输入值相同在任何时候必须产生相同的输出值
  2. 函数输入和输出值和其他隐藏信息和状态无关，也和I/O设备产生的外部输出无关
  3. 不能有语义上可观察的副作用，比如触发事件，修改了全局变量，参数，外部的存储等等

个人理解优点：相当于完全独立出来，不与其他部分相关联，安心使用它的功能就行 ，不容易产生bug

严格模式

  • 通过抛出错误来消除一些原有的静默错误
  • JS代码在执行的时候可以进行更多的优化，不用处理特殊的不规范的懒散代码
  • 禁用了ECMAScript未来版本中可能会定义的一些语法

想在哪开严格模式就在开头写”use strict”（一般js代码开头开启整个JS严格模式）

class和module自动开启严格模式

严格模式常见限制：

  1. 无法意外的创建全局变量（例如在函数里没用var创建一个变量）
  2. 抛出静默错误，感觉有点像c语言把wall警告改成error的感觉
  3. 函数参数不能有相同名字
  4. 不允许0开头的八进制语法
  5. 不允许使用with
  6. eval不再为上层引用变量
  7. this如果指向的不是对象，不再自动转换为对象类型，严格模式下this不绑定Window

ES6函数增强

默认参数

ES6可以在定义函数时定义默认参数，不传参数自动成默认值

注意事项：

  1. 默认参数放最后，放在剩余参数的前面
  2. 有默认参数的形参不计算在length内

展开运算符

可以运用在数组，字符串（可以当作字符数组看），创建字面量时能…快速复制以前对象的内容，但是不能迭代（可以进行浅拷贝,，ES9之后）

Symbol

ES5字符串才能作为key，到了ES6Symbol也能作为key，Symbol用来生成一个独一无二的值，避免想添加一个新属性给对象而此对象之前有重复key的情况

Set,Map

Set内部数据不会重复，类似C++stl，有增删改查的方法

WeakSet的内部数据也是不重复的，内部只能放对象类型，而且是弱引用，WeakSet不能遍历，没法直接获取中的对象（开发中用的比较少）

弱引用：要是只有弱引用引用一块内存空间的话，那么GC就可以把这块内存回收掉

Map映射基本和C++stl一样，也可迭代，可用for……of遍历

WeakMap也是弱引用，key也必须是对象，不能forEach，for……of遍历，具体作用以后响应式学

Proxy

监听对象属性的所有操作

Vue2的响应式原理：

缺点：描述符的设计初衷就不是用来监听的，监听内容很有限，比如新增、删除属性就不能监听

Proxy是代理对象，所有操作都在代理对象上，由代理对象监听

Proxy四个参数：

  • target：监听的对象
  • property：被设置/获取的属性key
  • value：新属性值
  • receiver：调用的代理对象

Proxy其他强大的监听用法

Reflect

Reflect是一个对象，解决什么东西都要加到Object使它太重，新加Reflect里都是操作对象的方法，好多用法相比于Object都更规范

  • 删除属性：Reflect.deleteProperty(obj, “name”)，返回bool类型

与Proxy结合使用

Promise

Promise主要解决公司内实现异步代码的困境，创建函数和调用函数不是同一个人的时候会很麻烦

Promise对象的成功和失败状态只会被更改一次和调用一次，动一次状态就锁死，后面再有reject()或resolve()都没用

异常处理

自己抛出异常用：throw new Error（）

自己捕获代码异常就不会抛给浏览器不影响后续代码执行

原型

基本对象原型

获取原型的方法：

obj.__proto__（存在兼容问题）

Object.getPrototypeOf（obj）

通过get方法获取属性的value时，没有就会从原型中查找

函数对象原型

函数看成普通对象是，是有__proto__这个隐式原型的

函数看成函数时，有prototype的（普通对象没有），可以foo.prototype这样用

new操作还有个作用：会把这个函数的显式原型的地址赋值给创建对象的隐式原型，也就是说对象改隐式原型也会同时改这个函数的显式原型，因为内存中只有一份

编写创建对象函数，将方法放在构造函数原型上时创建出来的对象将共享这个方法，这样不写在函数里能有效提高效率，写在函数里时每创建一个对象相当于就创建了一个一模一样的方法，浪费！

要是对原型对象修改太多，就不如重写原型

找不到的属性就从它的原型中找，一直这样找直到原型为null，所以就想到整个程序的对象和原型就构成了一个由双亲指针形成的树的结构，树根（Object 的 显式原型）的原型就是null，一个原型链就是树上的一条边

所以Object类是所有类的父类

原型继承

  1. 方式一：将子类的原型直接指向父类的原型：不能这么写，虽然实现了继承，但是全部原型都指在了一块，随便改一处会影响所有
  2. 方式二：创建个父类的实例对象，子类原型指向这个对象，实现继承并解决了上面问题，但是实例化相当于把继承来的属性写死了
  3. 方式三：借用构造函数，用父亲的构造函数显式绑定this实现

所以可以结合方式二和方式三实现继承（但是仍有一点点小问题：1. 构造函数至少会调用两次 2. 拥有两份属性 3. 先创建个实例随便赋上属性值？这种写法很别扭）

所以方法继承直接定义一个对象，这个对象原型指向父类的显式原型，并将它赋值给子类的显式原型

最终方法：寄生组合式继承

原型总结：

  • Function，Object，Foo都是Function的实例对象
  • 原型对象默认创建时，隐式原型都是指向Object的显示原型的，Ocbject指向null
  • Object是Function，其他类的父类

ES6原型

这里定义类就变成class了，比ES5方便很多（但是前面ES5是他的本质，相当于现在的写法是个语法糖），由于和C++及其相似，就只记录语法不一样的地方乐

  1. 构造方法只有一个，不能像C++重载多个，差评
  2. 子类构造方法用super记得放在this之前
  3. 子类重写父类方法时要是是在父类方法上追加功能的话，可以用super.父类方法()调用，然后写追加的功能
  4. JS只能继承一个类

babel

此处设计到工程化，暂时跳过~

开发规范：以下划线开头的变量只在内部访问不在外部访问

let，const

let类似var的使用

const声明变量是常量，但如果是引用类型，则可以改内存中的值（相比C++中指针常量，常量指针，这个好理解多了）

注意事项：

  1. let和const声明的变量名字不能相同
  2. let和const声明的变量不会添加到Window对象（因为虽然在全局环境上记录了，但是没有在对象环境上记录）
  3. let和const有块级作用域，但是函数仍然可以在之后访问
  4. 开发规范：有先使用const，确定要修改再用let

模板字符串

非常类似shell编程，可能都是脚本语言的原因

可以在函数后用，作为函数参数：foo`my name is ${name}`，在插入其他变量的模板字符串参数，整个参数会被拆分，第一个元素是数组，被模板字符串拆分的字符串组合，后面是一个个模板字符串传入的内容${}

将来会在react中经常应用，把css写进js中能直接拼接js的变量
